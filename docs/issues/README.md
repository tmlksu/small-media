# Known Issues and TODOs

This directory tracks known issues, limitations, and planned improvements.

## Issue Format

Each issue file should follow this format:

```markdown
# ISSUE-NNN: Title

## Status
Open | In Progress | Resolved | Won't Fix

## Priority
Critical | High | Medium | Low

## Description
What is the issue?

## Steps to Reproduce (if applicable)
1. ...
2. ...

## Expected Behavior
What should happen?

## Actual Behavior
What actually happens?

## Workaround (if any)
Temporary solution until fixed.

## Resolution (when resolved)
How was it fixed?
```
